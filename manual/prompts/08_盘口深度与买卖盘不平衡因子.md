# 盘口深度与买卖盘不平衡因子

> 提示：以下内容来自 future/期货因子，很多依赖股指成分股或期货的 Level2 盘口字段。迁移到股票时，只有在有五档/十档盘口快照时才能直接实现；如果只有普通 bar 数据，应改写为价差、振幅、VWAP 偏离、成交额冲击等代理思路。

## 核心含义

盘口类因子关注尚未成交的买卖力量：

- 买盘总量和卖盘总量谁更强。
- 买卖盘深度形状是否对称。
- 买一卖一价差是否扩大。
- 盘口不平衡的波动、偏度和持续性。
- 买盘价格变化和卖盘价格变化谁更不稳定。

这类因子预测周期通常较短，更适合分钟或更高频，不适合粗暴做日频裸因子。

## 计算方法与含义

### 总买卖盘不平衡

参考 `MinuteIndexBidAskRatioSharpe.py`：

```text
imbalance = (TotalBidVol - TotalAskVol) / (TotalBidVol + TotalAskVol)
factor = rolling_mean(imbalance, n) / rolling_std(imbalance, n)
```

含义：买盘挂单量相对卖盘挂单量是否持续占优。Sharpe 化后强调稳定不平衡，而不是瞬时尖峰。

股票转换：需要 Level2。日级可聚合为全天均值、尾盘均值、最大值、偏度。没有盘口时可用 `close_location` 或 `vwap_location` 弱代理。

### 买卖盘总量均值差

参考 `MinuteIndexBidAskTotVolMA20DiffRatio.py`：

```text
bid_sum = sum(TotalBidVol across stocks or levels)
ask_sum = sum(TotalAskVol across stocks or levels)
factor = rolling_mean((bid_sum - ask_sum) / (bid_sum + ask_sum), 20)
```

含义：整体买盘厚度与卖盘厚度的差异。买盘厚可能代表支撑，卖盘厚可能代表压力；但也可能是诱导挂单，需要结合成交确认。

股票转换：单股票可直接用盘口总量；横截面可比较股票之间的盘口不平衡 rank。

### 五档订单不平衡

参考 `MinuteIndexOrderImbalanceSkew.py`, `MinuteIndexOrderImbalanceStd.py`：

```text
bid_depth = BidV0 + BidV1 + BidV2 + BidV3 + BidV4
ask_depth = AskV0 + AskV1 + AskV2 + AskV3 + AskV4
oi = (bid_depth - ask_depth) / (bid_depth + ask_depth)
factor_skew = rolling_skew(oi, n)
factor_std = rolling_std(oi, n)
```

含义：盘口买卖力量的不平衡程度、波动程度和偏态。偏度高说明不平衡偶尔极端偏向一侧。

股票转换：可聚合成日级 `oi_mean`, `oi_std`, `oi_skew`, `oi_last_30min_mean`。

### 深度形状

参考 `MinuteIndexAskBidDepthRatio.py`, `MinuteIndexAskBidDepthRatioStd.py`：

```text
bid_slope = BidP0 / BidP4 - 1
ask_slope = AskP4 / AskP0 - 1
depth_ratio = bid_slope / ask_slope
factor = rolling_mean(depth_ratio, n)
```

含义：买盘从买一到买五的价格深度和卖盘从卖一到卖五的价格深度是否对称。形状不对称可能说明一侧挂单更“虚”或价格支撑更弱。

股票转换：需要多档盘口价格。可用十档扩展，或只用一档价差做简化。

### 买卖盘价格波动差

参考 `MinuteIndexBidAskRtnStdRatio.py`：

```text
bid_price = weighted_average(BidP0...BidP4, BidV0...BidV4)
ask_price = weighted_average(AskP0...AskP4, AskV0...AskV4)
bid_std = rolling_std(pct_change(bid_price), n)
ask_std = rolling_std(pct_change(ask_price), n)
factor = (bid_std - ask_std) / (bid_std + ask_std)
```

含义：买盘价格和卖盘价格哪一侧更不稳定。买盘更不稳定可能代表承接价格频繁变化，卖盘更不稳定可能代表抛压价格频繁变化。

股票转换：适合高频短周期预测，不建议直接作为长周期日频 alpha。

### 价差在上涨和下跌时的差异

参考 `MinuteIndexUpDownSpreadRatio.py`：

```text
spread = ask_price - bid_price
ret = close / delay(close, 1) - 1
factor = rolling_mean(spread where ret < 0, n) /
         rolling_mean(spread where ret > 0, n)
```

含义：下跌时价差是否显著扩大。如果下跌时流动性恶化，风险更高。

股票转换：有盘口时直接做；没有盘口时可用 `(high-low)/close` 作为粗略流动性压力代理。

## 股票优先实验模板

```text
order_imbalance = (sum_bid_volume_5 - sum_ask_volume_5) / (sum_bid_volume_5 + sum_ask_volume_5)
oi_mean_30m = mean(order_imbalance over last_30_minutes)
oi_std_day = std(order_imbalance over day)
spread_ratio = (ask1 - bid1) / mid_price
down_up_spread_ratio = mean(spread where ret < 0) / mean(spread where ret > 0)
```

优先级：有 Level2 时先做 `order_imbalance`、`spread_ratio`、尾盘盘口不平衡；没有 Level2 时不要硬做，改用 K 线和成交额冲击代理。
