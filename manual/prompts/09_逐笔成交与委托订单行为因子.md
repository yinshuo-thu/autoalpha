# 逐笔成交与委托订单行为因子

> 提示：以下内容来自 future/期货因子，很多依赖主动买卖成交、唯一订单数、大中小单、超级大单、新增挂单等逐笔或委托数据。迁移到股票时，必须确认是否有逐笔成交、逐笔委托、主动买卖方向、订单编号或 Level2 统计字段；普通 OHLCV 无法直接复刻。

## 核心含义

逐笔和委托行为类因子关注真实交易行为，而不是 bar 汇总：

- 主动买入和主动卖出谁更强。
- 大单和小单的资金结构。
- 订单是集中在少数大单，还是分散在许多小单。
- 买卖不平衡是否持续。
- 新增买卖挂单是否偏向一侧。

这类因子在股票上通常很有价值，但数据门槛最高，也更容易受交易成本和短周期噪声影响。

## 计算方法与含义

### 主动买卖成交量比

参考 `MinuteIndexBuySellRatioSharpe.py`：

```text
bs_ratio = BuyTradeQuantity / SellTradeQuantity - 1
weighted_bs_ratio_t = weighted_cross_section_mean(bs_ratio_i, weight_i)
factor = rolling_mean(weighted_bs_ratio, n) / rolling_std(weighted_bs_ratio, n)
```

含义：主动买量相对主动卖量是否持续占优。Sharpe 化后代表稳定的买卖力量，而不是单点冲击。

股票转换：单股票可做：

```text
bs_ratio = buy_trade_amount / sell_trade_amount - 1
factor = rolling_mean(bs_ratio, n)
```

如果没有主动买卖方向，可用 Lee-Ready 或 tick rule 估计，但误差要评估。

### 买卖资金不平衡自相关

参考 `MinuteIndexBuySellRatioAutoCorrSharpe.py`：

```text
bs_money = (BuyTradeMoney - SellTradeMoney) / (BuyTradeMoney + SellTradeMoney)
auto_corr = rolling_autocorr(bs_money, lag=1, window=n)
factor = rolling_mean(auto_corr, m) / rolling_std(auto_corr, m)
```

含义：买卖不平衡是否持续。如果自相关高，说明资金方向有延续性；如果低或负，说明买卖冲击很快反转。

股票转换：适合分钟级或逐笔聚合，不宜直接日频裸用。可聚合成尾盘买卖不平衡持续性。

### 唯一订单数与成交笔数比例

参考 `MinuteIndexUniqueBuyRatioMA10.py`, `MinuteIndexUniqueSellRatioMA10.py`, `MinuteIndexBSUniqueDiff30Ma.py`：

```text
buy_unique_ratio = BuyUniqueOrderNum / BuyTradeNum
sell_unique_ratio = SellUniqueOrderNum / SellTradeNum
factor = rolling_mean(sell_unique_ratio, n) - rolling_mean(buy_unique_ratio, n)
```

含义：衡量成交是否由大量独立订单驱动。唯一订单比例高，说明订单更分散；低则可能是少数订单反复成交或大单拆分。

股票转换：需要订单编号或交易所统计字段。若没有唯一订单数，可用笔均成交额、成交笔数、单笔金额分布替代。

### 大单小单结构

参考 `MinuteIndexSmallBigOrderDiff.py`, `MinuteIndexBuySellSuperorderCountDiff.py`：

```text
big_order_imbalance =
    (buy_big_order_money - sell_big_order_money) /
    (buy_big_order_money + sell_big_order_money)

small_order_imbalance =
    (buy_small_order_money - sell_small_order_money) /
    (buy_small_order_money + sell_small_order_money)

factor = big_order_imbalance - small_order_imbalance
```

含义：大资金和小资金方向是否一致。大单买、小单卖可能代表机构吸筹；大单卖、小单买可能代表派发或追涨。

股票转换：需要按单笔金额分桶。建议按股票自身历史成交额分位定义大单，而不是用固定金额阈值。

### 每唯一订单金额

参考 `MinuteIndexHighLowMoneyPerUniqueReturnDiff.py`, `MinuteIndexMoneyUniqueConsistency.py`：

```text
buy_money_per_order = BuyTradeMoney / BuyUniqueOrderNum
sell_money_per_order = SellTradeMoney / SellUniqueOrderNum
factor = rolling_mean(buy_money_per_order - sell_money_per_order, n)
```

含义：买方和卖方订单集中度。买方单均金额更大，可能代表更强买盘；卖方单均金额更大，可能代表大额抛压。

股票转换：可用笔均主动买入金额和笔均主动卖出金额替代。

### 新增买卖挂单差

参考 `MinuteIndexNetNewBidAskDiffSkew.py`：

```text
new_bid = diff(total_bid_volume) + sell_trade_quantity
new_ask = diff(total_ask_volume) + buy_trade_quantity
net_new_order = new_bid - new_ask
factor = rolling_skew(net_new_order, n)
```

含义：估计新增买卖挂单谁更强，并看其分布偏态。新增买单极端偏多或新增卖单极端偏多，代表订单簿压力。

股票转换：需要盘口快照和逐笔成交同时存在。没有盘口变动无法准确估计新增挂单。

### 按交易行为分组后的收益差

参考 `MinuteIndexHLBigSmallOrderCountRatioRetDiff.py`, `MinuteIndexHLTurnoverBSBigOrderCountRatioDiff.py`：

```text
behavior_ratio_i = buy_big_order_count_i / sell_big_order_count_i
rank_i = cross_section_rank(behavior_ratio_i)
factor = mean(ret_i where rank_i high) - mean(ret_i where rank_i low)
```

含义：大单买卖结构处于高分组的股票是否更强。原始期货因子常把成分股分组后聚合预测指数或期货。

股票转换：可直接作为个股横截面因子：大单买卖比 rank 越高，信号越强；也可做行业内分组。

## 股票优先实验模板

```text
active_bs_amount = (active_buy_amount - active_sell_amount) / (active_buy_amount + active_sell_amount)
active_bs_sharpe_20 = rolling_mean(active_bs_amount, 20) / rolling_std(active_bs_amount, 20)
big_order_imbalance = (big_buy_amount - big_sell_amount) / (big_buy_amount + big_sell_amount)
small_big_diff = big_order_imbalance - small_order_imbalance
avg_buy_order_size = active_buy_amount / active_buy_order_count
avg_sell_order_size = active_sell_amount / active_sell_order_count
order_size_diff = avg_buy_order_size - avg_sell_order_size
```

优先级：如果有逐笔数据，先做主动买卖不平衡、大单小单差、尾盘主动买卖不平衡。
