# AutoAlpha Research Program (v2 Real Data Engine)

## 1. Research Objective
This program orchestrates an **Autonomous Real-Data Formulaic Alpha Pipeline** optimized for 15-minute stock data. Its objective is to discover, validate, and select cross-sectional and time-series factors that consistently predict the competition `resp` while surviving turnover and concentration gates.

## 2. No Mock Policy Constraint
All data generated by this program MUST originate from the actual 1-minute `basic_pv` parquet files mapped to the active `SCIENTECH_DATA_ROOT` (default: `/Volumes/T7/data/eq_data_stage1`). 
The `resp` and `trading_restriction` variables are **strictly isolated** and ONLY accessed by the `Metrics Evaluator` post-construction. Any formula AST utilizing these fields will immediately trigger a `Compliance Guard` rejection.

## 3. Evolutionary Search
Instead of static template guessing, the planner:
- Retrieves the top-N "Submission Ready" factors from the leaderboard.
- Merges their ASTs using Crossover.
- Mutates functions (`ts_mean` -> `ts_std`) and parameters (10 -> 20 bars).
- Proposes offspring back to the Evaluator.

## 4. Factor Families
We categorize successful mutations into:
- **Mean Reversion**: Reversing sudden intra-day price dislocations (e.g., VWAP spreads).
- **Momentum**: Trend continuation across contiguous bars.
- **Liquidity/Volume**: Standardized volume surges highlighting institutional activity.
- **Volatility**: Ranges and standard deviations indicating risk profiles.

## 5. Experiment Execution Steps
1. The **Data Hub** caches all 15-minute aggregated data covering 2022-2024.
2. The **Formula Miner (EA)** proposes mathematical AST expressions using whitelisted operators (see `operator_catalog.py`).
3. The **Compliance Guard** tests bounds, coverage, structure, and leakages.
4. The **Evaluator** computes Information Coefficient (IC), Rank IC, Information Ratio (IR), and Turnover Series.
5. The **Strategy Lab** optionally trains ensemble models and simulates standard backtests (Long/Short Drawdown & Sharpe).
6. Successes are added to `leaderboard.json` and artifact logs.
