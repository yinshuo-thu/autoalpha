**分析日期：** 2026-04-20  
**数据来源：** 3 个已提交因子的官方平台回测指标

## 2. 平台评分公式（已验证）

### 2.1 质量门（任一不满足 → score = 0）

| 门控 | 条件 |
|------|------|
| 覆盖率 | `cover_all = 1` |
| 预测力 | `IC > 0.6` |
| 稳定性 | `IR > 2.5` |
| 换手率 | `tvr < 400` |
| 集中度 | `maxx < 50`，`\|minn\| < 50`，`max < 20`，`\|min\| < 20` |

> IC 的单位是"mean daily correlation × 100"（如 IC=2.13 表示日均相关系数 0.0213）。  
> tvr 的单位见平台文档：每 bar 换手 = Σ|Δα| / Σ|α|，日换手 = 当日各 bar 之和，tvr = 全天平均。

### 2.2 评分公式

$$
\text{score} = \left(\text{IC} - 0.0005 \times \text{tvr}\right) \times \sqrt{\text{IR}} \times 100
$$

- IC 奖励预测力，tvr 项作为简化交易成本惩罚，√IR 奖励信号一致性。

### 2.3 公式验证（误差 < 0.001）

| 因子 | IC | IR | tvr | 公式计算 score | 平台返回 score | 误差 |
|------|-----|-----|-----|--------------|--------------|------|
| alpha_001 | 2.1324 | 5.5736 | 55.6497 | **496.8581** | 496.8581 | 0.000043 ✅ |
| alpha_007 | 1.6654 | 24.2608 | 691.6246 | 0（tvr 失败） | 0.0 | 0.0 ✅ |
| alpha_006 | 1.7912 | 28.2852 | 714.9574 | 0（tvr 失败） | 0.0 | 0.0 ✅ |

**验证过程（alpha_001）：**
```
step 1: IC − 0.0005 × tvr = 2.1324 − 0.0005 × 55.6497
                           = 2.1324 − 0.027825
                           = 2.104575

step 2: √IR = √5.5736 ≈ 2.360847

step 3: score = 2.104575 × 2.360847 × 100 ≈ 496.858
```

## 3. alpha_006 / alpha_007 评分归零原因

两因子唯一失败的门控均为 `tvr < 400`：

| 因子 | tvr（平台） | 门控结果 |
|------|------------|---------|
| alpha_006 | 714.96 | ❌ 超出限制（需 < 400） |
| alpha_007 | 691.62 | ❌ 超出限制（需 < 400） |

这两个因子均基于每 bar 内部的开高低收比值，每 bar 都会大幅重新排列，导致相邻 bar 之间的权重变化极大，换手率天然偏高。相比之下，EMA-based 的 alpha_001 信号变化平滑，tvr 仅 55.65。

## 4. 本地指标 vs 平台指标的系统性偏差

### 4.1 偏差数据

| 因子 | 指标 | 本地（submission_like） | 平台 | 比值（平台/本地） |
|------|------|----------------------|------|----------------|
| alpha_001 | IC  | 2.3418 | 2.1324 | 0.91 |
|           | IR  | 6.2907 | 5.5736 | 0.89 |
|           | tvr | 44.73  | 55.65  | **1.24** |
| alpha_006 | IC  | 1.5214 | 1.7912 | 1.18 |
|           | IR  | 29.017 | 28.285 | 0.97 |
|           | tvr | 637.98 | 714.96 | **1.12** |
| alpha_007 | IC  | 1.4366 | 1.6654 | 1.16 |
|           | IR  | 26.369 | 24.261 | 0.92 |
|           | tvr | 618.61 | 691.62 | **1.12** |

### 4.2 偏差原因分析

**tvr 偏差（最关键）：**

本地计算方式（`evaluator.py: calc_turnover_submission_wide`）：
```python
weights_bps = alpha / |alpha_total| × 10_000
daily_tvr = mean_per_day(|Δweights_bps|) / 10
```

平台计算方式（官方文档）：
```
tvr(t) = Σᵢ |αᵢ(t) − αᵢ(t−1)| / Σᵢ |αᵢ(t)|
daily_tvr = sum over all bars in a day
tvr = average over all trading days
```

两者的差异来源：
1. **聚合方式**：本地用 `mean/10`，平台用 `sum`（每日各 bar 之和）
2. **归一化时机**：本地先把 alpha 转成 bps 权重再求差，平台直接对原始 alpha 做差后归一化
3. **分母随时间变化**：本地分母用当前 bar，平台分母也用当前 bar，理论相同，但实现细节可能不同

综合效果：**本地 tvr 系统性低估平台 tvr 约 12%~24%**。

**IC/IR 偏差：**

IC 偏差方向不固定（alpha_001 本地偏高，alpha_006/007 本地偏低），可能原因：
- 平台使用的交易限制（trading restriction）mask 与本地数据有细微差异
- 平台可能在某些日期剔除了额外的股票（如停牌、异常等）
- 数据对齐细节不同（如第一个 bar 的处理）

## 5. 结论与操作建议

### 5.1 分数公式（已精确验证）

$$
\boxed{\text{score} = (\text{IC} - 0.0005 \times \text{tvr}) \times \sqrt{\text{IR}} \times 100}
$$

本地用此公式计算与平台完全一致，可以完全信赖。

### 5.2 换手率风险（重要！）

由于本地 tvr **系统性低估** 平台 tvr 约 12%~24%，保守策略：

| 本地 tvr | 平台预期 tvr | 建议 |
|---------|------------|------|
| < 330   | < 400（大概率通过） | ✅ 可以提交 |
| 330~360 | 接近 400 边界 | ⚠️ 谨慎，建议降低换手 |
| > 360   | 大概率超过 400 | ❌ 不建议提交 |

### 5.3 提升 score 的方向

根据公式 `score = (IC − 0.0005 × tvr) × √IR × 100`：

1. **最重要**：控制 tvr < 330（本地）— tvr gate 是最容易踩中的陷阱
2. **其次**：提升 IC — 直接线性加分
3. **再次**：提升 IR — 通过 √IR 乘数放大，IC 高且 IR 高的因子回报最大
4. **注意**：tvr 惩罚系数只有 0.0005，IC=2 时 tvr 需超过 4000 才会使括号内归零，因此在通过 tvr gate 的前提下，适当增加换手以换取更高 IC 是合理的
